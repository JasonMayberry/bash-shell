 Symbolic notation is divided into three parts.
   â€¢ Who the change will affect.
   â€¢ Which operation will be performed.
   â€¢ What permission will be set.

                 To specify who is affected, a combination of the characters 
                      â€œuâ€, â€œgâ€, â€œoâ€, and â€œaâ€ is used as shown in Table.
         +--------+-----------------------------------------------------------------+
         | Symbol |                         Meaning                                 |
         +--------+-----------------------------------------------------------------+
         |   u    |  Short for â€œuserâ€ but means the file or directory owner.        |
         |   g    |  Group owner.                                                   |
         |   o    |  Short for â€œothersâ€ but means world.                            |
         |   a    |  Short for â€œall.â€ This is the combination of â€œuâ€, â€œgâ€, and â€œoâ€. |
         +--------+-----------------------------------------------------------------+

If no character is specified, â€œallâ€ will be assumed. The operation may be a â€œ+â€ indicating that 
a permission is to be added, a â€œ-â€ indicating that a permission is to be taken away, 
or a â€œ=â€ indicating that only the specified permissions are to be applied and that all others 
are to be removed.
Permissions are specified with the â€œrâ€, â€œwâ€, and â€œxâ€ characters. 

This Table provides some examples of symbolic notation:
+------------+--------------------------------------------------------------------------------------------+
| Notation   |                          Meaning of Symbolic Notation                                      |

# Signals "KILL"
The kill command is used to â€œkillâ€ processes.
  [me@linuxbox ~]$ xlogo &
  [1] 28401
  [me@linuxbox ~]$ kill 28401
  [1]+  Terminated              xlogo
  We first launch xlogo in the background. The shell prints the jobspec and the PID of the 
  background process. Next, we use the kill command and specify the PID of the process we 
  want to terminate. We could have also specified the process using a jobspec 
  (for example, %1) instead of a PID.
When a process is terminated with kill, it is given no opportunity to â€œclean upâ€ after 
itself or save its work. For this reason, the KILL signal should be used only as a last 
resort when other termination signals fail.

The kill command is used to send signals to programs. 
Its most common syntax looks like this:
kill [-signal] PID...
Example: $ kill -1 13546  OR  $ kill -INT 13601
